# 玄学小程序首页结构稿

## 页面目标

- 让用户一打开就愿意直接提问。
- 让用户感受到系统很强，但不需要先学习体系。
- 把“智能管家”放在绝对视觉中心。

## 首页定位

- 这不是分类页。
- 这不是体系介绍页。
- 这是一个会接话、会分配体系、会继续追问的首页。

## 首屏结构

### 1. 顶部状态区

- 左侧：品牌符号或星图印记
- 中间：一句很短的标题
- 右侧：历史记录 / 我的入口

建议标题：
- 玄学智能管家
- 今晚想问什么

建议副文案：
- 直接说问题，系统会自动分配合适的玄学体系

### 2. 主视觉区

- 深色宇宙背景
- 很轻的金线星图或罗盘纹理
- 中间一个半透明发光核心
- 不做重舞台，不做满屏神像

作用：
- 给神秘感
- 给高级感
- 不抢输入框

### 3. 智能管家提问框

- 首页最重要模块
- 尺寸足够大，像一张会发光的任务卡
- 允许直接输入自然语言
- 输入框下方带“系统会如何处理”的轻提示

建议占位文案：
- 直接说你现在最想问的事
- 例如：这套房子适不适合长期住
- 例如：我今年要不要换城市发展

输入框下方辅助提示：
- 自动识别命盘 / 风水 / 时机 / 关系 / 择日 / 塔罗

### 4. 管家反馈预告区

- 在正式提问前，先让用户知道结果形态
- 以 3 个简短标签卡呈现

建议标签：
- 自动分配体系
- 缺失信息会追问
- 给出统一结论

### 5. 快捷起算入口

- 作为辅助入口存在
- 放在提问框下面
- 用 6 个图标卡片

建议入口：
- 看命盘
- 看风水
- 看时机
- 看关系
- 看择日
- 看塔罗

要求：
- 卡片要克制，不做廉价宫殿按钮
- 图标感强于文字感

### 6. 最近记录

- 只露出 1 到 2 条
- 作用是让用户知道这不是一次性产品

每条记录展示：
- 问题摘要
- 时间
- 本轮参与体系数量

### 7. 底部导航

- 首页
- 管家
- 风水
- 记录
- 我的

## 页面节奏

1. 先看见神秘氛围
2. 立刻看见输入框
3. 知道系统会自动分配
4. 再看到快捷入口
5. 最后看到历史记录

## 视觉关键词

- 墨黑
- 黛青
- 暗金细线
- 雾面玻璃
- 轻发光
- 深空感
- 玉色高光

## 禁忌

- 不要大段世界观文案
- 不要首页先让用户选复杂分类
- 不要堆满符咒、龙凤、宫殿、神像
- 不要廉价赛博紫
- 不要把输入框做小

## 首页一句话原则

- 用户进来后，第一反应应该是：我现在就可以直接问。
