# 玄学系统全量测试报告（2026-06-16）

## 结论先说

- 本轮按 `21` 个已实现体系逐个提问，标准 benchmark 结果：`21 / 21 可回答`
- 全量回归测试结果：`python -m unittest discover -s tests -v` -> `143 OK`
- 但“能回答”不等于“完全没问题”
- 当前整体状态：
  - `命盘类 / 时机类 / 起名类`：整体较稳
  - `风水 / 择日 / 阴阳道`：能答，但部分回答仍偏粗筛
  - `少数真人超短口语问法`：之前有漏接，已修掉一批

## 三、我作为严格测试者，当前判定出的真实问题

### P1：择日与六爻的“missingInputs”提示和实际算出来的结果不完全一致
- 现象：
  - `择日` 题里已经写了 `地点在上海`，诊断仍报 `缺地点`
  - `六爻/梅花` 已经能按数字起算，但诊断仍报 `缺起卦方式或卦象`
- 影响：
  - 用户表面上拿到了答案，但总控层会显得不够自洽

### P1：风水回答现在多半还是“朝向粗筛”，不是完整实盘判断
- 现象：
  - 能答“偏适合长期居住”
  - 但多数仍停在 `坐北朝南 -> 粗判断`
- 影响：
  - 真用户会觉得“有答，但不够深”

### P2：阴阳道回答能给结论，但解释层仍偏短
- 现象：
  - 能说 `偏不利，建议绕开或改期`
  - 但为什么不利、怎么绕、替代方案是什么，还不够展开

### P2：部分体系仍有轻微重复表达
- 典型：
  - `姓名学`
  - `卡巴拉`
  - `人类图`
- 影响：
  - 不影响可用性，但会显得回答打磨感不够

### P2：婚恋/婚姻类问题，部分命盘体系更偏事业解读，关系线不够强
- 典型：
  - `紫微`
  - `吠陀`
- 影响：
  - 用户问“婚姻和事业”，现在很多时候还是事业答得更实

## 四、这一轮的最终结论

如果你问我一句最直白的话：

- `大多数体系已经能答，而且不是瞎答`
- `命盘类、时机类、起名类目前最稳`
- `风水、择日、阴阳道现在是“能用，但还没到很老练”`
- `总控诊断层还有少量自洽性问题，需要继续抠`

如果下一轮继续做，我建议优先压这 4 个方向：

1. `每个体系再补 2 条更像真人的口语题`
2. `风水从朝向粗筛升级到户型/楼栋/层数/人宅匹配的更完整回答`
3. `修掉 missingInputs 与 computable 冲突`
4. `把婚恋题在紫微 / 吠陀里的关系线答得更实`
